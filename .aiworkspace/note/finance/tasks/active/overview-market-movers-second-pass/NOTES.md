# Notes

- Browser-session auto refresh should run only the selected Market Movers coverage, not all intraday jobs.
- Existing local run history shows Top1000 quote-fast snapshots around 6.5-7.1 seconds and Top2000 around 12.5 seconds, but provider timeout / fallback can stretch much longer.
- Momentum comparison should be framed as context for trend persistence, not as a buy/sell recommendation.
- Volume Rank should not sort the already return-ranked Top N subset. It now ranks a separate `volume_rows` frame over the returnable universe.
- Daily volume means the current stored intraday snapshot / EOD day. Weekly / monthly / yearly volume means the current return window's average daily volume / average daily dollar volume plus total period volume / dollar volume.
- Top2000 yearly still spends most time resolving the eligible date window, not aggregating volume. The new volume aggregation itself is roughly 1 second locally after index-forced symbol/date reads.
- Catalyst Links are a manual investigation launchpad, not a catalyst classifier. The service builds a row model from selected symbol, name, period, coverage, rank type, and rank; the UI only renders those outbound rows.
- Catalyst Links intentionally do not fetch Yahoo / Google / SEC / IR pages from the UI. Search URLs are prebuilt so users can inspect sources manually in a new browser context.
- Why It Moved V1.5 keeps the same manual-investigation boundary but adds the selected mover's existing row context before outbound research. It is not an automatic cause detector and should not label a movement as earnings, filing, macro, analyst, or sentiment driven.
- Compact metadata lookup is button-triggered for one selected ticker only. Selectbox changes rebuild the read model and outbound links, but do not call yfinance, SEC, or any external source.
- Compact news metadata stores only title, source / publisher, published time, and URL in current Streamlit session state. Compact SEC metadata stores only form, filing date, title / description, and URL. Article bodies and filing bodies are intentionally excluded.
- Session-only was chosen because V1.5 is an investigation aid with provider fragility and no durable evidence review policy yet. DB-backed compact metadata belongs to a later V2 only after retention, source quality, freshness, and replay semantics are approved.
- Review follow-up: Compact metadata has four terminal display states after a lookup: `OK`, `PARTIAL`, `FAILED`, and `NO_METADATA`. `PARTIAL` means at least one compact metadata table has rows while another provider failed; it should be shown as warning-level investigation evidence, not as a complete lookup.
- Review follow-up: The six external search destinations are not primary workflow buttons anymore. They live in a collapsed `External searches` table with `Open` links, including Korean Google News / Naver News rows, so the primary panel action remains `Fetch compact metadata`.
- UX review: Why It Moved is still prototype-level from a product UX standpoint. It now has safer state handling and less button noise, but it still lacks a purpose-built investigation layout such as source-status summary, evidence grouping, freshness cues, prioritized next checks, and compact side-by-side News / SEC comparison.
- Prototype-exit guideline: The next UX/UI pass should keep the same no-classifier boundary but reshape the panel into an investigation board: movement summary header, provider status strip, News / SEC / External Searches sections with clear empty / partial / failed states, and optional Korean-source lane as outbound-only until a Korean metadata provider policy is approved.
- V1.6 implemented the prototype-exit guideline without changing provider scope. `build_market_mover_metadata_status_strip` is the Streamlit-free status contract for `NOT_REQUESTED`, `OK`, `PARTIAL`, `FAILED`, and `NO_METADATA`; `PARTIAL` is warning tone and should not be rendered as complete success.
- V1.6 display order is fixed as title / boundary caption, ticker selector, movement summary header, metadata status strip, `Fetch compact metadata`, and `Investigation Leads`. The status strip shows lookup status, News row count or failure, SEC row count or failure, fetched timestamp, and `Session-only` storage boundary.
- Investigation Leads uses stacked `News Metadata`, `SEC Filings`, and collapsed `External Searches` sections. The service payload still uses `URL`, but UI display tables map it to a clickable `Open` column. SEC filing display stays form / filing date / title / open and sorts newest date first with form priority inside ties (`8-K`, `10-Q`, `10-K`, `S-1`, `S-3`, `S-8`, `4`, others).
- Korean copy pass: The main `Why It Moved` title and source names can remain English, but board labels, captions, empty states, status values, fetch button, and URL link display text should be Korean in the visible UI. Contract status keys such as `PARTIAL` remain English internally.
