# Risks

- Score weights can look more precise than the available evidence. UI copy must frame scores as Final Review decision support, not return prediction.
- Open review count must not become a hidden blocker unless route/cap policy explicitly says so.
- Existing saved decisions and registry rows must remain compatible.
- Automatic weakness optimization, new portfolio variant generation, and improved portfolio backtest comparison are intentionally out of scope for this task.
- Current score caps are explicit policy heuristics. They should be revisited after real saved Final Review rows accumulate.
