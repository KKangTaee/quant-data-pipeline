# Plan

## 이걸 하는 이유?

Overview Market Intelligence 1차 prototype은 기능 흐름을 만들었다. 2차는 정식화 baseline이므로 구현자가 “무엇을 확인해야 다음 차수로 넘어갈 수 있는지”를 문서만 보고 재현할 수 있어야 한다.

## Scope

- 2차 acceptance checklist를 phase 문서에 고정한다.
- Market Movers, Sector / Industry, Events, Ingestion refresh의 manual QA route를 정리한다.
- closeout 전 필수 command와 browser smoke 기준을 명시한다.

## Done Criteria

- phase folder에 2차 acceptance checklist가 있다.
- runbook에서 refresh state, missing diagnostics, event freshness 확인 방법을 알 수 있다.
- 검증 command와 남은 3차 한계가 문서화된다.
