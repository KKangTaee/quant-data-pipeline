# Risks

- stale snapshot을 missing gap과 동일하게 직접 수집하면 공식 source가 없는 symbol에서 불필요한 실패가 생길 수 있다.
- Final Review에서 수집까지 직접 실행하면 저장 evidence 확인 stage와 Level2 보강 stage가 섞인다.
- raw audit 문자열 번역이 새 수치 판정을 만들어서는 안 된다. 기존 저장값의 표현과 action routing만 소유한다.

## Closeout

- stale symbol은 기존 official / bridge collection plan 안에서만 실행 대상으로 만들고, source가 없는 경우의 실패 의미는 Practical Validation이 소유한다.
- Final Review는 수집을 실행하지 않고 같은 후보의 2단계 보강 화면으로 이동시키는 것으로 stage 혼합을 방지했다.
- 새 audit id가 추가되면 사용자 언어 mapping과 action taxonomy 계약을 함께 보강해야 한다.
