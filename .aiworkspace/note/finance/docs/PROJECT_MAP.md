```text
Backtest Analysis
  -> Practical Validation
```

м—ӯн• :

- Backtest AnalysisлҠ” нӣ„ліҙ sourceлҘј л§Ңл“ лӢӨ.
- Practical ValidationмқҖ sourceлҘј мӢӨм „ нҲ¬мһ… м „ мЎ°кұҙмңјлЎң кІҖмҰқн•ҳкі  source traits кё°л°ҳ module plannerлЎң н•„мҲҳ / мЎ°кұҙл¶Җ / нӣ„мҶҚ м°ёкі  кІҖмҰқкіј Final Review мқҙлҸҷ gateлҘј л§Ңл“ лӢӨ. нҷ”л©ҙмқҖ `нӣ„ліҙ Source нҷ•мқё -> кІҖмҰқ кё°мӨҖ м„Өм • / мӢӨм „ мһ¬кІҖмҰқ мӢӨн–ү -> кІҖмҰқ кІ°лЎ  / лӢӨмқҢ н–үлҸҷ -> кІҖмҰқ кё°мӨҖ мғҒм„ё`мқҳ 4к°ң user-facing flowлЎң мқҪлҠ”лӢӨ. Flow 1мқҖ source snapshotкіј Backtestм—җм„ң л„ҳм–ҙмҳЁ 2м°Ё нҷ•мқё н•ӯлӘ©л§Ң нҷ•мқён•ҳкі , Flow 2лҠ” validation profileкіј replay мӢӨн–үмқ„ н•Ёк»ҳ м„Өм •н•ңлӢӨ. Flow 3мқҖ workspace read modelмқҳ Practical Validation outcome summaryлҘј нҶөн•ҙ `нҶөкіј / ліҙк°• нӣ„ мһ¬кІҖмҰқ / мӢӨм „ мӮ¬мҡ© м–ҙл ӨмӣҖ`, мӢӨнҢЁ м№ҙн…Ңкі лҰ¬, кІҖмҰқ м№ҙн…Ңкі лҰ¬, Final Review мқҙлҸҷ к°ҖлҠҘ м—¬л¶Җ, `кІҖмҰқ кІ°кіј м ҖмһҘ(кё°лЎқмҡ©)`, `м ҖмһҘн•ҳкі  Final ReviewлЎң мқҙлҸҷ` CTAлҘј compactн•ҳкІҢ ліҙм—¬мЈјлҠ” first-read conclusion / next-action surfaceлӢӨ. PV-owned REVIEW cautionмқҙ лӮЁм•„лҸ„ м°ЁлӢЁмқҙ м•„лӢҲл©ҙ Flow 3мқҖ `мЈјмқҳ нҸ¬н•Ё мқҙлҸҷ к°ҖлҠҘ`мңјлЎң н‘ңмӢңн•ҙ л…ёлһҖл¶Ҳ нҶөкіјмҷҖ мҙҲлЎқ нҶөкіјлҘј кө¬л¶„н•ңлӢӨ. ReactлҠ” CTA intentл§Ң м „лӢ¬н•ҳкі , save-only audit append / Final Review handoffлҠ” Python page + service pathк°Җ мІҳлҰ¬н•ңлӢӨ. `кІҖмҰқн•ң кІғ / н•ҙкІ°н•ҙм•ј н•  н•ӯлӘ© / н•ҙкІ° л°©лІ• / нҶөкіј кё°мӨҖ / мң„м№ҳ` к°ҷмқҖ н•ҙкІ° guideлҠ” Flow 4м—җм„ң нҷ•мқён•ңлӢӨ. Flow 4мқҳ `н•ҙкІ° л°©лІ•`мқҖ `resolution_guide.action_steps` лІҲнҳё лӘ©лЎқмңјлЎң л ҢлҚ”л§Ғн•ҙ rowлі„ `Next Action`, provider ліҙк°•, DB ліҙк°•, Flow 2 мһ¬кІҖмҰқ к°ҷмқҖ мӢӨн–ү лӢЁкі„лҘј л¶„лҰ¬н•ҙ ліҙм—¬мӨҖлӢӨ. Flow 4лҠ” к°ҷмқҖ read modelмқҳ `visible_criteria_detail_groups`лҘј `м№ҙн…Ңкі лҰ¬лі„ кІҖмҰқ кІ°кіј` boardлЎң ліҙм—¬мЈјкі , Source & Replay / Data Quality / Bias Control / Comparison Validity / Realism / Tradability / Validation Method Strength / Stress / Robustness / Portfolio Construction / Conditional Evidenceлі„ `мғҒнғң / нҶөкіјн•ң кё°мӨҖ / лӮЁмқҖ л¬ём ң / нҢҗм •`мқ„ мҡ”м•Ҫн•ңлӢӨ. мқҙ boardлҠ” raw status мң„м—җ `нҶөкіј`, `ліҙк°• нӣ„ мһ¬кІҖмҰқ н•„мҡ”`, `мӢӨм „ мӮ¬мҡ© м–ҙл ӨмӣҖ` main outcome layerлҘј л‘”лӢӨ. `READY`лҠ” нҶөкіјлЎң мқҪкі , `REVIEW`лҠ” н•ҳлӮҳмқҳ Final Review мҲҷм ңк°Җ м•„лӢҲлқј `лҚ°мқҙн„° мЈјмқҳ`, `2лӢЁкі„ мӢӨмҡ©м„ұ мЈјмқҳ`, `мөңмў… нҢҗлӢЁ м°ёкі `, `Monitoring м¶”м Ғ`, `м ҖмһҘ м „ ліҙк°•` roleлЎң л¶„лҰ¬н•ңлӢӨ. м Ғмҡ©лҗң Practical Validation categoryк°Җ REVIEW-onlyлқјлҸ„ Flow 4м—җ н‘ңмӢңн•ҳкі , Final Review / Monitoring м „мҡ© referenceлҠ” actionable issueк°Җ м•„лӢҲлқј ліҙмЎ° м°ёкі лЎң лӮ®м¶ҳлӢӨ. `NEEDS_INPUT` / `NOT_RUN`мқҖ ліҙк°• нӣ„ мһ¬кІҖмҰқ лҢҖмғҒ, `BLOCKED`лҠ” нҳ„мһ¬ мғҒнғңлЎң мӢӨм „ нӣ„ліҙ мӮ¬мҡ©мқҙ м–ҙл Өмҡҙ м°ЁлӢЁ кІ°лЎ мңјлЎң л¶„лҰ¬н•ңлӢӨ. `selected_route_preflight`мҷҖ Final Review м ҖмһҘ м „ gapмқҖ Flow 3 CTA disabled reasonкіј Flow 4 ліҙмЎ° к·јкұ°м—җм„ң лӢӨлЈЁл©° лі„лҸ„ visible Flow 5 containerлЎң л°ҳліөн•ҳм§Җ м•ҠлҠ”лӢӨ. Stress / robustness лҜёмӢӨн–үмқҖ кё°ліё review, construction riskлҠ” ETF-like лҳҗлҠ” weighted mixм—җл§Ң м Ғмҡ©, sentiment risk-on/off overlayлҠ” context-onlyлЎң л‘”лӢӨ. `NEEDS_INPUT` / `NOT_RUN` к°ҷмқҖ raw statusлҠ” кё°мҲ  detailм—җ лӮЁкё°л©°, `кІҖмҰқ кІ°кіј м ҖмһҘ(кё°лЎқмҡ©)`мқҖ audit trailл§Ң лӮЁкё°кі  Gate лҜёнҶөкіј resultлҠ” Final Review нӣ„ліҙк°Җ м•„лӢҲлӢӨ.
- Final ReviewлҠ” Practical Validation GateлҘј нҶөкіјн•ң resultл§Ң source pickerм—җ н‘ңмӢңн•ңлӢӨ. Provider / Look-through / Robustness Lab / Construction Risk / Risk Contribution / Component Role Weight / Validation Method Strength / Data Coverage / Backtest Realism к·јкұ°мҷҖ investability packetмқ„ мқҪм–ҙ profile-aware gate policyлЎң selected-route к°ҖлҠҘ м—¬л¶ҖлҘј нҢҗм •н•ңлӢӨ. Investment report scorecardлҠ” `Investment`, `Risk`, `Readiness`, `Evidence Quality`, `Monitoring Suitability` weighted dimensionкіј Level2 REVIEW roleлі„ score impactлҘј ліҙм—¬мЈјл©°, hard blocker / selected-route not-ready / gate review-required / excessive open reviewлҠ” score capмңјлЎң м¶”мІңк¶Ң кіјлҢҖнҸүк°ҖлҘј л§үлҠ”лӢӨ. Investment report first-readлҠ” мҙқнҸү л°”лЎң м•„лһҳ `м„ұкіј н•ҙм„қ / мң„н—ҳ н•ҙм„қ / к·јкұ° мӢ лў°лҸ„ / Monitoring м Ғн•©м„ұ` 4н–үмқ„ л‘җкі , named evidence adapterлЎң нҢҗм •н•ң 10к°ң pattern мӨ‘ мӨ‘мҡ”лҸ„ мғҒмң„ мөңлҢҖ 6к°ңл§Ң `нҳ„мһ¬ м§„лӢЁ / мқҳлҜё / ліҖнҷ” мЎ°кұҙ / лӢӨмқҢ н–үлҸҷ`мңјлЎң ліҙм—¬мӨҖлӢӨ. лӮҳлЁём§Җ нҢЁн„ҙкіј source / кё°мӨҖмқј / technical pathлҠ” м ‘нһҢ мғҒм„ё к·јкұ°м—җ лӮЁлҠ”лӢӨ. Level2 REVIEWлҠ” `мөңмў… нҢҗлӢЁм—җм„ң кІ°м •н•  кІғ / 2лӢЁкі„м—җм„ң мқёмҲҳн•ң м ңн•ңмӮ¬н•ӯ / MonitoringмңјлЎң л„ҳкёё мЎ°кұҙ / м„ м • м „ н•ҙмҶҢн•  м°ЁлӢЁ н•ӯлӘ©`мңјлЎң мҶҢмң к¶Ңмқ„ лӮҳлҲ„л©°, Flow4 ліҙк°• л¬ёкө¬лҘј Final Review мӮ¬мҡ©мһҗ н–үлҸҷмңјлЎң л°ҳліөн•ҳм§Җ м•ҠлҠ”лӢӨ.
