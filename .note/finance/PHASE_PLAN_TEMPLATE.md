# Phase Plan Template

## 이 문서는 무엇인가
- 이 phase 문서가 어떤 역할을 하는지 짧게 적는다.

## 목적
- 이 phase에서 이루고 싶은 핵심 목적 1~3개를 적는다.

## 쉽게 말하면
- 이 phase를 비전문가도 따라올 수 있게 짧게 풀어쓴다.
- "지금 왜 이걸 하고 있는지"가 바로 보이게 적는다.

## 왜 필요한가
- 이전 phase까지의 한계나 현재 병목을 적는다.
- 왜 지금 이 phase가 필요한지 설명한다.

## 이 phase가 끝나면 좋은 점
- 이 phase가 끝났을 때 사용자/운영자/개발자 입장에서 무엇이 좋아지는지 적는다.

## 이 phase에서 다루는 대상
- 전략 family, 기능 범위, UI 영역, 문서 범위 등
  이번 phase가 직접 다루는 범위를 적는다.

## 현재 구현 우선순위
1. 작업 이름
   - 쉽게 말하면:
   - 왜 먼저 하는가:
   - 기대 효과:
2. 작업 이름
   - 쉽게 말하면:
   - 왜 필요한가:
   - 기대 효과:

## 이 문서에서 자주 쓰는 용어
- phase 안에서 반복해서 나오는 어려운 용어를 짧게 적는다.
- 여러 문서에서 계속 나올 표현이면 glossary에도 추가한다.

## 이번 phase의 운영 원칙
- 구현 우선 / 검증 우선 / 문서 우선 같은 현재 운영 원칙을 적는다.
- 어떤 검증까지만 하고, 무엇은 일부러 뒤로 미루는지 적는다.

## 이번 phase의 주요 작업 단위
- 이 phase에서 진행할 큰 작업 단위들을 적는다.
- 각 작업 단위마다
  - 무엇을 바꾸는지
  - 왜 그 작업이 필요한지
  - 작업이 끝나면 무엇이 좋아지는지
  를 함께 적는다.
- 가능하면 `slice` 같은 내부 용어보다
  `작업 단위`, `첫 번째 작업`, `두 번째 작업`
  처럼 바로 이해되는 표현을 쓴다.

## 다음에 확인할 것
- 다음 확인 포인트나 다음 작업 단위에서 볼 항목을 적는다.

## 한 줄 정리
- 이 phase를 한 문장으로 요약한다.
