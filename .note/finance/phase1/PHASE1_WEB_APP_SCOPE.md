# Phase 1 Web App Scope

## 목적
이 문서는 내부용 웹 앱 1차 개발 범위를 확정하기 위한 기준 문서다.

이 단계의 목적은:
- 처음부터 많은 기능을 넣지 않고
- 가장 필요한 운영 기능만 먼저 구현하고
- 이후 확장 가능한 최소 구조를 확보하는 것이다.

---

## 1차 목표

1차 웹 앱의 목표는 아래 한 문장으로 정리한다.

**OHLCV / fundamentals / factors 적재를 버튼으로 실행하고, 실행 결과를 바로 확인할 수 있는 내부 운영용 웹 앱을 만든다.**

---

## 포함 범위

### 1. 기본 웹 앱 실행
- 로컬에서 실행 가능한 내부용 웹 앱
- 브라우저에서 접근 가능

### 2. 핵심 수집 실행 버튼
- OHLCV 수집 실행
- 요약 재무제표 수집 실행
- 팩터 계산 실행

### 3. 최소 파라미터 입력
- symbols
- freq
- start / end 또는 period
- interval

단, 1차에서는 꼭 필요한 파라미터만 노출한다.

### 4. 실행 결과 표시
- 작업명
- 성공/실패 상태
- 시작/종료 시각
- 처리 건수
- 실패 심볼 수
- 요약 메시지

### 5. 최소 운영 확인 기능
- 최근 실행 결과 1건 이상 표시
- 실패 시 에러 메시지 또는 실패 심볼 일부 표시

---

## 제외 범위

### 1. 백테스트 UI
- 전략 입력 화면
- 백테스트 결과 화면
- 성과 그래프 UI

### 2. 복잡한 데이터 관리 UI
- 상세 재무제표 조회 화면
- 자산 프로필 상세 편집 화면
- 테이블 브라우저

### 3. 고급 운영 기능
- 사용자 로그인
- 권한 관리
- 멀티유저 지원
- 원격 배포 환경 최적화

### 4. 자동화
- 예약 실행
- 백그라운드 워커
- 큐 시스템

### 5. 복잡한 프론트엔드 구조
- React 기반 별도 프론트엔드
- API 서버와 프론트 분리

## 1차 기능 우선순위

### P1
- 앱 실행
- OHLCV 수집 버튼
- fundamentals 수집 버튼
- factors 계산 버튼
- 결과 표시

### P2
- 최근 실행 이력 표시
- 로그 요약 표시
- 실패 심볼 요약

### P3
- asset profile 수집 버튼
- detailed financial statements 수집 버튼
- 전체 파이프라인 실행 버튼

1차 구현은 P1 완료를 기준으로 한다.

---

## 성공 기준

아래 조건을 만족하면 1차 범위는 성공으로 본다.

1. 웹 앱이 로컬에서 실행된다.
2. 버튼 클릭으로 OHLCV 수집이 실행된다.
3. 버튼 클릭으로 fundamentals 수집이 실행된다.
4. 버튼 클릭으로 factors 계산이 실행된다.
5. 각 실행 결과가 화면에서 확인된다.
6. 실패 시 최소한 어떤 작업이 실패했는지 확인 가능하다.

---

## 기술 방향

1차에서는 아래 방향을 고정한다.

- UI: Streamlit
- 실행 계층: wrapper job 함수
- 수집 로직: 기존 `finance/data/*` 함수 재사용
- 문서화: `.note/finance` 기준
