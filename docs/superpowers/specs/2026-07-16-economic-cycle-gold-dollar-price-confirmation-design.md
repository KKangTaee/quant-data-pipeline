# 경제 사이클 금·달러 가격 확인 설계

## 문제

현재 `금·달러` 카드는 미국 경제의 `회복 국면`을 카드 제목처럼 표시한다.
사용자는 이를 금과 달러 가격 자체가 회복 중이라는 뜻으로 읽을 수 있다.
또한 거시경제 근거만으로 `우호`를 판정해 실제 가격이 하락하는 상황을 설명하지 못한다.

## 목표

- 경제 전체 국면과 개별 자산 가격 흐름을 명확하게 분리한다.
- 금과 달러를 서로 다른 카드로 분리한다.
- 거시경제 배경과 실제 가격 흐름이 같은지 별도로 확인한다.
- 두 정보가 충돌하면 결론을 강제로 합치지 않고 `불일치`로 표시한다.

## 검토한 접근

### 1. UUP 대리 ETF

- 장점: 기존 가격 테이블을 그대로 사용할 수 있다.
- 단점: 로컬 자료가 2026-06-26에서 멈춰 있으며 ETF 추적 오차가 있다.
- 결론: 현재 가격 확인의 기본 원천으로 사용하지 않는다.

### 2. 통화선물 합성 달러 지표

- 장점: 기존 FX futures를 사용할 수 있다.
- 단점: 자체 가중치를 만들어야 하고 공식 DXY와 달라진다.
- 결론: 불필요한 모델 복잡성 때문에 사용하지 않는다.

### 3. 달러인덱스 선물 직접 저장

- 장점: ICE Dollar Index futures를 직접 추적하고 금 futures와 동일한 저장 경계를 사용한다.
- 단점: `DX-Y.NYB` instrument와 최초 일봉 backfill이 필요하다.
- 결론: 이 방식을 채택한다.

ICE는 [DX](https://www.ice.com/products/194/US-Dollar-Index-Futures)를 미국 달러와 주요 통화 바스켓의 가치를 나타내는 US Dollar Index futures로 정의한다.
앱에서는 기존 yfinance pilot source의 continuous symbol `DX-Y.NYB`를 사용하며 exchange-grade realtime 데이터로 표현하지 않는다.

## 데이터 경계

### 가격 원천

| 카드 | provider symbol | 표시 이름 | 저장 위치 |
|---|---|---|---|
| 금 | `GC=F` | 금 선물 | `finance_price.futures_ohlcv` |
| 달러 | `DX-Y.NYB` | 달러인덱스 선물 | `finance_price.futures_ohlcv` |

- `finance/data/futures_market.py`의 core instrument에 달러인덱스 선물을 추가한다.
- 기존 daily futures action이 5년 일봉을 수집하도록 한다.
- UI와 economic-cycle service는 provider를 직접 호출하지 않고 DB만 읽는다.
- 가격 기준일은 각 symbol의 최신 저장 일봉 날짜다.
- 경제 배경 기준일과 가격 기준일을 카드에 따로 표시한다.
- 최신 가격이 7일 이상 오래됐으면 가격 판단은 `자료 부족`으로 제한한다.

## 가격 흐름

각 자산에 대해 거래일 기준 수익률을 계산한다.

- 최근 1주: 5거래일
- 최근 1개월: 21거래일
- 최근 3개월: 63거래일

1주는 사용자가 체감하는 최근 움직임을 설명하는 보조 정보다.
가격 확인 상태는 노이즈를 줄이기 위해 1개월과 3개월로 판정한다.

| 조건 | 가격 확인 |
|---|---|
| 1개월과 3개월 모두 `+1%` 초과 | 상승 확인 |
| 1개월과 3개월 모두 `-1%` 미만 | 하락 확인 |
| 그 외 | 방향 혼재 |
| 64개 미만 또는 stale | 자료 부족 |

연속선물에는 roll 효과가 포함될 수 있음을 방법론에 표시한다.

## 경제 배경

### 금

- 실물·고용 약화: 우호
- 금융·선행 여건 강화: 부담
- 물가·정책 압력 강화: 우호

### 달러

- 실물·고용 강화: 우호
- 금융·선행 여건 강화: 부담
- 물가·정책 압력 강화: 우호

기존과 동일하게 우호와 부담 개수 차이가 두 개 이상일 때만 한 방향으로 판정하고, 나머지는 `혼재`로 둔다.
경제 국면은 점수에 다시 넣지 않는다.

## 종합 판단

| 경제 배경 | 가격 확인 | 종합 판단 |
|---|---|---|
| 우호 | 상승 확인 | 배경과 가격 일치 |
| 부담 | 하락 확인 | 배경과 가격 일치 |
| 우호 | 하락 확인 | 배경과 가격 불일치 |
| 부담 | 상승 확인 | 배경과 가격 불일치 |
| 혼재 또는 방향 혼재 | 모든 상태 | 혼재 |
| 가격 자료 부족 | 자료 부족 | 가격 확인 대기 |

종합 판단은 수익률 전망이나 매수·매도 신호가 아니다.

## UI

### 공통 문구

- 기존 `회복 국면`을 `경제 국면: 회복`으로 변경한다.
- 섹션 보조 문구는 `경제 배경과 실제 가격을 분리해 확인`으로 변경한다.

### 금·달러 카드

1. 카드 상단: `금` 또는 `달러`, `경제 국면: 회복`
2. 종합 배지: `일치 / 불일치 / 혼재 / 가격 확인 대기`
3. 경제 배경: `우호 / 부담 / 혼재 / 자료 부족`
4. 가격 확인: `상승 확인 / 하락 확인 / 방향 혼재 / 자료 부족`
5. 가격 수익률: `1주 / 1개월 / 3개월`
6. 기준일: `경제 2026-06-30 · 가격 YYYY-MM-DD`
7. 주요 거시 근거 두 개
8. 바뀌는 조건

### 다른 자산 카드

- 채권·금리, 주식, 원자재는 기존 거시경제 배경 판정을 유지한다.
- 상단 문구만 `경제 국면: ...`으로 명확하게 바꾼다.
- 별도 가격 원천 연결은 이번 범위에 포함하지 않는다.

### 배치

- desktop/tablet: 2열, 마지막 원자재 카드는 전체 너비
- mobile: 1열

## 오류와 제한 상태

- 달러인덱스 instrument 또는 일봉이 없으면 달러 카드는 유지하고 가격 확인만 `자료 부족`으로 표시한다.
- 금 자료 부족도 동일하게 처리한다.
- 가격 누락이 경제사이클 확률이나 다른 자산 카드 렌더링을 막지 않는다.
- 경제 배경과 가격 기준일이 다르다는 사실을 숨기지 않는다.

## 테스트

- 금과 달러가 별도 asset group으로 생성된다.
- 가격 1주·1개월·3개월 수익률과 threshold 경계가 검증된다.
- 경제 배경과 가격의 일치/불일치 표가 검증된다.
- 가격 부족·stale 상태가 다른 카드를 막지 않는다.
- economic-cycle service가 DB-only 가격 loader를 호출한다.
- React source contract, production build, 실제 DB read model을 검증한다.
- desktop/mobile에서 카드 배치, overflow, 날짜와 상태 가독성을 확인한다.

## 제외 범위

- 금·달러 가격 예측
- 목표가격 또는 매매 추천
- intraday/realtime feed
- 경제사이클 확률 모델 또는 publication gate 변경
- 채권·주식·원자재 가격 확인 추가
