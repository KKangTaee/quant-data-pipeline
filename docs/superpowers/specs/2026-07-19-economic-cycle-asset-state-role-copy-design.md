# 경제사이클 자산별 공통 경제 배경 역할별 문구 설계

## 배경

Economic Cycle의 `현재와 전망의 판단 근거`는 실물 수준과 전망 영향을 역할별 문구로 구분한다. 그러나 `자산별 확인 포인트`의 `관측된 경제 상태`는 같은 네 factor를 여전히 공통 `강화 / 약화 / 중립`으로 표시한다. 이 때문에 금융·선행 여건의 양수 방향과 물가·정책 압력의 양수 방향이 모두 좋은 신호처럼 읽히고, 한 화면 안에서 같은 데이터가 서로 다른 의미로 설명된다.

## 목표

- 자산별 카드에서도 네 factor를 Evidence와 같은 역할별 언어로 표시한다.
- 이 정보가 특정 자산의 상태가 아니라 모든 자산 카드에 공유되는 미국 경제 배경임을 명확히 한다.
- 현재 실물 수준과 1·2개월 전망에 주는 영향을 한 문단 안에서 분리한다.
- 모델 점수, 임계값, 방향 enum, 확률과 자산별 경로 계산은 유지한다.

## 사용자 화면

### 제목

- 기존: `관측된 경제 상태`
- 변경: `사이클 판단의 공통 경제 배경`

### 요약

요약은 두 역할을 순서대로 설명한다.

1. 현재 수준: 생산·소비 활동과 고용·소득이 자기 과거 기준보다 높은지, 낮은지, 부근인지 설명한다.
2. 전망 여건: 금융·선행 여건과 물가·정책 압력이 향후 1·2개월 전망을 지원하는지, 부담을 주는지 설명한다.

현재 예시 값이 실물 두 factor 음수, 전망 두 factor 양수라면 다음처럼 표시한다.

> 현재 수준: 생산·소비 활동과 고용·소득은 자기 과거 기준 이하입니다. 전망 여건: 금융·선행 여건은 전망을 지원하지만, 물가·정책 압력은 부담을 주는 방향입니다.

자료 부족 factor는 누락하지 않고 해당 역할 안에서 `자료 부족`으로 표시한다.

### 배지

| factor | 양수 방향 | 음수 방향 | 중립 | 결측 |
| --- | --- | --- | --- | --- |
| 생산·소비 활동 | 기준 이상 | 기준 이하 | 기준 부근 | 자료 부족 |
| 고용·소득 | 기준 이상 | 기준 이하 | 기준 부근 | 자료 부족 |
| 금융·선행 여건 | 전망 지원 | 전망 부담 | 영향 중립 | 자료 부족 |
| 물가·정책 압력 | 전망 부담 | 부담 완화 | 영향 중립 | 자료 부족 |

색상은 문구의 의미를 보조한다. 실물 기준 이상과 전망 지원·부담 완화는 지원 계열, 실물 기준 이하와 전망 부담은 부담 계열, 중립·자료 부족은 중립 계열을 사용한다. 색상만으로 의미를 전달하지 않는다.

## 데이터와 코드 경계

`finance/economic_cycle_asset_pathways.py`의 `_economic_state()`는 기존 `direction`과 `value`를 유지하면서 역할별 요약 문장을 만든다. payload 구조와 방향 enum은 바꾸지 않는다.

React의 `EconomicStateBlock`은 factor와 direction을 받아 Evidence와 동일한 역할별 배지 문구와 tone으로 변환한다. 가능하면 역할별 표현 규칙을 작은 helper로 분리해 summary와 badge의 의미가 달라지지 않게 한다.

이 블록은 자산마다 같은 `economic_state`를 반복해서 받지만, 제목에서 `공통 경제 배경`임을 밝혀 특정 자산 자체가 강화되거나 약화된다는 오해를 줄인다.

## 변경하지 않는 범위

- factor 계산과 expanding robust scale
- `±0.15` 방향 임계값
- 현재·1개월·2개월 국면 확률
- 자산별 가격, 금리, EPS, 원자재 경로 판단
- 매매 신호나 가격 원인 추론

## 검증

- Python 테스트에서 역할별 summary와 기존 direction enum 보존을 확인한다.
- React source contract에서 새 제목과 네 종류의 역할별 배지 문구를 확인하고 이전 공통 제목을 제거한다.
- Economic Cycle focused regression과 React production build를 실행한다.
- 실제 Streamlit 화면을 desktop과 420px에서 확인하고 텍스트 겹침, 가로 overflow, console error를 점검한다.

## 완료 조건

- 같은 네 factor가 Evidence와 자산별 카드에서 같은 의미로 읽힌다.
- 사용자가 `금융·선행 여건 강화`를 무조건 좋은 상태, `물가·정책 압력 강화`를 물가 안정으로 오해할 표현이 남지 않는다.
- 모델과 데이터 계약을 바꾸지 않고 표시와 summary 해석만 일관되게 정리된다.
