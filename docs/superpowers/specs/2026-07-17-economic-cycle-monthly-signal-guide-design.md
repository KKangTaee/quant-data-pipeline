# Economic Cycle Monthly Signal Guide Design

## 목적

사용자가 월별 경제사이클 신호를 장기 경기확정이나 즉시 행동 지시로 오해하지 않고, 매달 무엇을 관찰하고 언제 준비 수준을 높일지 화면 안에서 다시 확인하게 한다.

## 위치와 동작

- `자산별 확인 포인트` 다음, `방법론과 품질` 앞에 배치한다.
- 네이티브 `details/summary`를 사용하고 기본 상태는 닫힘이다.
- 제목은 `월별 사이클 신호 활용법`, 보조 문구는 `관찰 → 준비 → 조정 검토`로 표시한다.
- 데이터 fetch, 세션 상태, 현재 국면 자동 강조를 추가하지 않는다.

## 펼친 내용

### 3단계 원칙

1. 한 달 신호는 `관찰`한다.
2. 같은 방향이 여러 달 이어지면 `준비`한다.
3. 실물·금융·가격이 함께 확인되면 기존 계획 안에서 `조정 검토`한다.

### 네 국면 카드

- 회복: 낮은 경제활동 수준에서 모멘텀이 개선되는 상태. 지속성과 금융여건 완화를 확인하고 관심 대상을 준비한다.
- 확장: 경제활동 수준과 모멘텀이 함께 강한 상태. 기업 이익과 시장 확산을 확인하고 물가·금리 부담을 점검한다.
- 둔화: 경제활동 수준은 높지만 모멘텀이 약해지는 상태. 집중도·레버리지·경기민감 노출을 점검하고 정상화와 침체 위험을 구분한다.
- 침체: 경제활동 수준과 모멘텀이 모두 약한 상태. 현금 필요와 손실 감내 범위를 점검하고 실물·신용 약화의 확산 여부를 확인한다.

### 전환 요약

- 침체 → 회복: 조기 전환 신호이며 대세상승장 확정이 아니다.
- 회복 → 확장: 유동성 기대에서 실제 이익 확인으로 무게가 이동한다.
- 확장 → 둔화: 정상화인지 광범위한 약화인지 구분한다.
- 둔화 → 침체: 약화의 폭과 지속성이 함께 커지는지 확인한다.

## 표현 원칙

- 투자 추천, 자산별 목표 비중, 수익률 예측을 제공하지 않는다.
- `매수`, `매도`, `주문` 문구를 사용하지 않는다.
- 각 국면은 방향을 설명하고 사용자가 확인할 항목을 제시한다.
- 국면별 색은 좌측 선이 아니라 카드 전체의 연한 배경으로 구분한다.
- expander 하단에 `월별 조기경보이며 NBER 공식 경기판정이나 행동 지시가 아니다`라는 경계를 둔다.

## 반응형

- 데스크톱: 3단계 원칙은 3열, 국면과 전환은 2열이다.
- 모바일: 모든 그리드를 1열로 재배치한다.
- summary와 긴 문장은 줄바꿈을 허용하고 가로 스크롤을 만들지 않는다.

## 완료 조건

- 경제 사이클 하단에 기본 닫힘 expander가 표시된다.
- 네 국면과 네 전환의 의미·대응을 간결하게 다시 읽을 수 있다.
- 분석 모델, payload, 확률 및 자산 해석 로직은 변경되지 않는다.
- source contract test, Vite build, 데스크톱·모바일 브라우저 QA를 통과한다.
